# Getting started

## Prerequisites

The video pipeline runs Manim as a subprocess, so you need a working Python + Manim install in addition to Node.

| Tool | Min version | Notes |
|------|-------------|-------|
| Node.js | 20.11+ | ESM `import.meta.dirname` is used |
| npm | 10+ | shipped with Node |
| Python | 3.10+ | for Manim |
| FFmpeg | any modern | Manim shells out for encoding |
| LaTeX | any | only used if a generated scene uses `MathTex`; the prompt biases away from LaTeX, but install for safety |
| Manim Community | 0.19+ | `pip install manim` |

### Quick deps check

```bash
node --version            # >= 20.11
python3 --version         # >= 3.10
ffmpeg -version | head -1 # any
manim --version           # Manim Community v0.19+
```

If `manim` isn't installed:

```bash
pip install --user manim
# Ubuntu/Debian extras for LaTeX (optional, scenes mostly avoid LaTeX):
sudo apt install texlive-latex-extra ffmpeg
```

## Install

```bash
git clone <repo>
cd ai-tutor
npm install
```

## Environment variables

Copy `.env.example` to `.env` and fill in:

```env
ANTHROPIC_API_KEY=sk-ant-...      # required
ANTHROPIC_MODEL=claude-sonnet-4-6 # optional, defaults to claude-sonnet-4-6
ANTHROPIC_SUMMARY_MODEL=          # optional, falls back to ANTHROPIC_MODEL
PORT=8787                         # optional, server port
```

`ANTHROPIC_API_KEY` is the only required value. Get one at [https://console.anthropic.com/](https://console.anthropic.com/).

`.env` is gitignored. So is `.env.save`, `.env.*.local`, `server/tmp/`, `public/videos/`, and `docs/manim/`.

## Run

The dev script starts both the Vite frontend and the Express API in parallel with prefixed logs:

```bash
npm run dev
```

You should see:

```
[web] VITE v8 ready in 200 ms
[web] Local: http://localhost:5173/
[api] tutor server on :8787
```

Open [http://localhost:5173/](http://localhost:5173/).

You can also run them separately:

```bash
npm run dev:web   # vite only
npm run dev:api   # tsx watch server/index.ts
```

## Production build (frontend only)

```bash
npm run build
npm run preview
```

The server is run via `tsx` in dev. There is no separate server build step — `tsx` executes TypeScript directly. To run the server in production, use `tsx server/index.ts` or compile with `tsc` and run the JS.

## First lesson

1. Open the app. Click **Upload PDF** and pick any document.
2. Wait for the **Indexed** badge to turn green (the document summary is generated by Claude — used to ground every lesson in the document's domain).
3. Highlight a sentence or paragraph. A popover appears.
4. Press <kbd>Enter</kbd> for an explanation, or <kbd>⌘ Enter</kbd> / <kbd>Ctrl Enter</kbd> for an animation.
5. The new lesson appears as a node on the right canvas and opens automatically in focus mode.

If the lesson type is `video_manim`, expect 20–60 seconds of rendering. A progress bar shows the four stages: planning → generating → rendering → done.

## Troubleshooting first run

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `tutor server on :8787` then immediate exit | port in use | `lsof -i :8787` and kill, or change `PORT` |
| `Manim render failed` | manim missing or LaTeX failure | run `manim --version`; if LaTeX-related, instruct prompt to avoid `MathTex` (already biased) or install `texlive-latex-extra` |
| Frontend blank | API not running | check `[api]` line in `npm run dev` output |
| Selection popover doesn't appear | text layer disabled | upload a real text PDF (not a scanned image without OCR) |
| `403`/`401` from Anthropic | bad or missing API key | check `.env` |

See [manim-pipeline.md](manim-pipeline.md) for video-specific debugging.
